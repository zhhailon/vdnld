# vdnld

`vdnld` is a Python CLI downloader inspired by studying Video DownloadHelper extension builds.

> Code generated by [Claude](https://claude.ai) and [OpenAI Codex](https://openai.com/codex).

## Layout

- `src/vdnld/cli.py`: argument parsing and entrypoint
- `src/vdnld/app.py`: orchestration (plan → print → execute)
- `src/vdnld/download/manager.py`: URL probing and download planning
- `src/vdnld/download/execute.py`: plan execution, ffmpeg integration, progress output
- `src/vdnld/download/direct.py`: direct media download with resume
- `src/vdnld/download/hls.py`: HLS segment download with resume
- `src/vdnld/download/cache.py`: per-target cache directory management
- `src/vdnld/extractors/registry.py`: URL-to-extractor routing
- `src/vdnld/extractors/youtube.py`: YouTube page extraction (progressive + HLS)
- `src/vdnld/extractors/browser.py`: Playwright-based browser extraction
- `src/vdnld/manifest/m3u8.py`: HLS master/media playlist parser
- `src/vdnld/merge/policy.py`: merge flag logic
- `src/vdnld/net/http.py`: HTTP fetch helpers
- `src/vdnld/dependencies.py`: runtime dependency checks

## Getting Started

```bash
uv run vdnld --help
uv run vdnld https://example.com/video
uv run vdnld https://example.com/video -o output.mp4
uv run vdnld https://example.com/master.m3u8 --quality 720p
uv run vdnld --browser https://example.com/page
uv run vdnld --interactive-browser https://example.com/page
uv run vdnld --interactive-browser --plan-only https://example.com/page
uv run vdnld --no-resume https://example.com/video
uv run vdnld --clear-cache https://example.com/video
uv run python -m unittest discover -s tests -v
```

Set `UV_CACHE_DIR=/tmp/uv-cache` to control where uv stores its cache (useful in WSL).

## CLI Flags

| Flag | Description |
|------|-------------|
| `url` | Source page or media URL |
| `-o / --output` | Output file path (auto-derived from title or URL if omitted) |
| `-q / --quality` | HLS quality for master playlists: `highest`, `lowest`, `720p`, or `1280x720` |
| `--browser` | Enable Playwright headless browser fallback |
| `--interactive-browser` | Open a visible browser for manual login/playback, then resume |
| `--plan-only` | Print the download plan and exit without running ffmpeg |
| `--no-resume` | Discard existing cache and start fresh |
| `--clear-cache` | Remove the cache for this target and exit |

## Current Capabilities

**Planning:**
- Routes YouTube and Vimeo URLs to site-specific extractors
- Probes generic URLs over HTTP
- Detects and parses HLS playlists (master and media M3U8)
- Selects the highest-bandwidth variant from a master playlist by default, with `--quality` override
- Falls back to browser extraction when static probing is insufficient

**YouTube extractor:**
- Extracts progressive download URLs from page HTML
- Extracts HLS manifest URLs from page HTML

**Browser extractor (Playwright):**
- Captures HLS and direct media requests made during page load
- Interactive mode waits for manual user interaction before capturing
- Forwards captured request headers (referer, origin, user-agent, cookie) to subsequent fetches

**Execution:**
- Downloads direct media to a local cache and remuxes with `ffmpeg -c copy`
- Downloads HLS segments to a local cache and remuxes with `ffmpeg -c copy`
- Resumes interrupted downloads from a `.<name>.vdnld/` cache directory
- Renders live ffmpeg progress (percentage bar when duration is known, elapsed time otherwise)
- Fails early with a clear error if `ffmpeg` is not on `PATH`
- Derives output filename from extracted media title, falling back to the URL path stem

## Runtime Requirements

`ffmpeg` must be on `PATH`:

```bash
which ffmpeg
ffmpeg -version
```

Playwright is required only when using `--browser` or `--interactive-browser`:

```bash
uv add playwright
uv run playwright install chromium
```
